# 汇编语言编程既严苛又优美，或许还是通向更强人工智能的路径

Section: Gregory Barber
Date: 2025-10-13T07:00:00-04:00
Rubric: 无论你的芯片在运行一款复古电脑游戏，还是最新的 DeepSeek 模型，它都倾向于回报你使用它的母语。

1999 年，《过山车大亨》并不是当时最时髦的电脑游戏。但如果你把像素翻开一看——那摇摇欲坠的设施、饥渴又晕头转向的人群（以及他们身后挥舞抹布清扫的清洁工）——深入到代码层面，你会看到一种近乎疯狂的匠心独运。该游戏的唯一开发者克里斯·索耶（Chris Sawyer）把整款作品都写在了汇编语言中。

某些编程语言，比如 Python、Go 或 C++，被称为“高级语言”，因为它们在一定程度上像人类语言，像是用命令和习语写成的，甚至像在诗歌朗诵会上能听懂的一套表达。一般而言，编译器会把这些内容转换成机器真正读取的内容：由 1 和 0（或十六进制）组成的数据块，告诉真实晶体管如何运作。汇编语言是“低级语言”中最底层的一层，它与机器的本地语言几乎一一对应。它是直接面向金属的编码。用汇编构建一款复杂的电脑游戏，就像用刨理过的猫毛去编织一幅挂毯。

为什么会有人这么做？我最近在他家乡苏格兰的索耶。他告诉我，效率是其中一个原因。20 世纪 90 年代，高级语言所需的一些工具还不够成熟。编译器非常慢。调试器也很差。索耶可以通过使用 x86 汇编——英特尔芯片的通用语言——来规避这些问题，走自己的路。

我们都知道那并不是真正的原因。真正的原因是热爱。在转向《过山车大亨》之前，索耶还用汇编写过另一款游戏《运输大亨》。那款游戏让玩家掌控一座城市的道路、铁路站点、跑道和港口。我曾想象索耶像一个模型铁路爱好者——一段一段地铺设轨道、手工缝合人造草皮，每一个细节都是选择，也是体力活。要把这些精心打磨的像素从位图推送到显示器，索耶必须把芯片的全部潜能都榨出来。“《过山车大亨》的诞生，完全是因为我熟悉了可能性所能触及的边界，”他对我说。

在当下这个数字资源极其充裕的时代，谈“在约束中工作”似乎有些异类——如今只需一次调用 AI 训练算法中的一个函数，就可能动用上百万张 GPU。使用汇编语言时，你得到的只有一件事，而且就是你所要求的那件事；哪怕你要求错了，它仍会照办——很多程序员都是用代价惨痛的方式学到这一点的。汇编正是如此地残酷，也同样迷人。它要求你精确表达自己的意思。

我对汇编的发明者们欠了个欠妥当的解释。他们想要的并不是把事情变难，而是让事情变简单。我想，他们早已厌倦了反复装打孔卡、在蒸汽朋克般的怪兽机器上来回拨动开关。也许他们曾梦想过一个像我们这样的世界：计算机只需极少的指令，就能做出如此多的事情。

第一门汇编语言由凯瑟琳·布斯（Kathleen Booth）在 20 世纪 40 年代创造（尽管她长期未得到应有的认可，真是见怪不怪），但它几乎不像“语言”。代码用来代替另一种代码。为了让机器执行某个操作——比如机器码中的“0,0111”——你得改用一连串字母和符号，然后通过一款新软件（汇编器）把它们翻译成二进制。随后，这些指令很快演化出更贴近人的助记符，例如“MOV”。

要理解汇编，前提是理解 CPU 本身——它能做什么，更关键是它做不到什么。一块芯片的物理设计、连接 AND 与 XOR 逻辑门的电路到底是如何布局的，决定了它的行为。它的功能其实很基础，把指令拆解成最基本的步骤：从内存取出某个值并放入一个临时仓位，也就是寄存器；在其中解码；执行一些运算，比如比较两个数值或将它们相加；再把结果写回内存。

随着芯片的发展，汇编语言也演化出了新的方言。让第一位登上月球的人类所使用的代码就是汇编——它是为一种芯片设计的，阿波罗11号导航计算机。若你想阅读 Furby 的泄露源代码，就需要精通 6502。要改写你的 TI-83 计算器程序，就得懂 z80。学习一种芯片的语言——比如英特尔的 x86——然后再转去用 Arm，就像在贝鲁特学阿拉伯语后，想在突尼斯或喀土穆勉强生活。祝你好运。

我在大学学过 x86 汇编，那时我像个数学难民。我的同学似乎很享受 Java 那些乏味的咒语般的写法，但我却爱上了汇编这门逻辑游戏。它很容易失败，但如果你去看电路和寄存器，就能明白失败的原因。每当我用这门不完全算是语言的简单指令写代码时，我都会感到自己技艺高超；也知道这种精通有多脆弱。你得把某些字节放在那里——不，是放那里，放到那个寄存器，放到那些电容里。记住这个。忘掉那个。就这样反复打磨，像逐个上色一样，一件一件地处理。

确实，汇编语言在日常编程工作里已经没那么多用处了。高级语言的效率如此之高，其抽象层通常几乎总是更可取。即便汇编语言的发明者后来也转向了别的领域；他的九十年代最后一篇论文之一，用神经网络把海豹的吠声与其身份匹配。Sawyer 也转向了别的方向。他最近在家居自动化上也有涉猎——照明、温度传感器、音响系统等等，用树莓派通过 Python 编写控制程序，他起初觉得这“相当拗口”，他跟我这样说过。但即便在那颗小处理器上，它也能把工作很好地做好。

然后 DeepSeek 这类事物出现了，再次提醒我们：人类仍然可以与硬件沟通得更直接。就在今年早些时候，这家制造了这些极高效 AI 模型的中国公司颠覆了“AI 进步只能靠更多芯片和更多能耗”这一叙事。汇编就是其中一个出人意料的原因。DeepSeek 的工程师深入 Nvidia 芯片的底层底板，指挥每一台独立机器在恰到好处的时刻把数据从 32 位压缩到 8 位——用牺牲精度换效率。围观者都惊呆了。你们真的能做到这一点吗？DeepSeek 的工程师们重新触碰了一门大多数人早已遗忘的艺术。

在 2023 年，DeepMind 的研究人员也做过类似的尝试，他们教会了一台机器 x86 汇编语言，然后要求它改进 C 语言中长期存在的 sort() 函数。AI 做出了怪异且反直觉的选择，在寄存器之间进行奇特的跳转，最终只精确地减少了一步。也许只省下了几分之一毫秒的时间，但现在这个新算法已被正式采用，每天都会发生成千上万次。

对我来说，这让我想起了我们这些人类创造了这些机器，即便它们看似已经陷入超出我们理解范围的复杂漩涡，它们仍然仍受我们掌控。我们总是可以让它们变得更好。这让我想起 Sawyer 在谈起他最近用 Raspberry Pi 做的家庭编程实验时说过的话。那可能只是他的想象，但他觉得屏幕有些卡顿。若可以，他会重新编写代码。但可惜的是，Sawyer 与机器并不使用同一种汇编语言。
