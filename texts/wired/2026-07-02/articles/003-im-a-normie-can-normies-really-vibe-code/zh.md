# 我算“普通人”。普通人真的能 vibe code 吗？

Section: Chris Colin
Date: May 18, 2026 6:00 AM

把我带进科技未来的那条狗“矮矮、壮壮的”。我妈妈在今年年初被它在城市公园里正面撞中之前，先看到的只是这个特点：又矮又厚、身体很结实，而且速度快到足以把她的右胫骨打断。但这不再展开讲她的事。我们来谈谈这件事给我人生带来的连锁反应：在我整整两年半都没真正学会编程之后，我很快就要开始做我的第一个软件开发项目了。

如果你也被一条“矮矮、壮壮”的狗撞断了妈妈的胫骨，你就会知道接下来一连串小小的尴尬和麻烦。比如我爸爸在电话树系统里来回打转、试着处理我妈妈医疗事宜的那些小时。从大局上看，这些让人抓狂的电话到底算不算大事？算不了。但那条笨狗偏偏在一个很“技术化”的时刻把事搞砸了。历史上第一次，问题不必非常严重，也能迫使我们动员起一整套严肃的工具。

只要硅谷开始兜售“零摩擦的明天”，我们这些普通人就一直是它的被动消费者——在应用商店里滑来滑去，指望总有人替我们做出想要的一切。直到 AI 和它那副“民主化”副驾驶登场：vibe coding。若这一承诺属实，那我们几乎可以零编程基础地，随心所欲地做出自己的一批小众且微不足道的应用。我们只要指出那些让我们烦恼的需求，一套由大型语言模型、代码生成器和开发环境组成的系统就会立刻开工。

小众且微不足道？那不就是我吗！当别人用 vibe code 做简历筛选、库存追踪、自动化助手来提高工作效率时，我心里盯着的目标完全不同。过去几年里，我在个人和职业层面都越来越困在一个叫作 sludge（“沉没负担”）的议题上：一股细碎行政义务的浪潮，它正越来越像是定义现代生活的方式，并不断侵蚀我们的行动能力。是处理保险，还是把保险和医生对接，或是追踪航空里程，或者在孩子的学校门户网站上办事。要对一笔账单提出异议，或取消你早已不记得自己还在用的某个流媒体订阅。每一项都像是对时间的一次独立袭击。可它们并不独立，而是从同一片菌根网络里冒出的不同蘑菇。

在某种程度上，这是一个校准问题。虽然更大的问题至少在理论上可能吸引人关注——立法、新闻报道、参议院听证会——但那些太小、没到可以起诉程度的问题，就只会变成生活的一部分。历史的大趋势或许会朝向正义弯曲，但当你去对抗一美元的银行手续费时，它却更像是朝着不断重复的等待音乐方向弯曲。

这正是“vibe coding”这种幻想吸引我的地方。这些麻烦并非复杂性的偶然副产物；它们往往本身就是设计好的功能。一个让人困惑的门户、一次掉线、一个故意模糊到让人想半途而废的流程。大规模放大后，它们的作用不太像是“漏洞”，更像是既定政策。 我设想的应用会把这种现象显露出来，让这些义务的累积负担更难被忽略。我想让你脑海中浮现的是一片颤抖的蘑菇田。

我妈妈虽然缺少一双健康的腿，但她却用一份 Claude Pro 订阅来弥补。过去几年我反复在她面前提起人工智能在环境、政治和经济上的影响，但最近一个周日我把这些都放在一边，开车去她家。聊了会儿她的胫骨问题后，我打开了她的电脑，开始“发 vibes”。

现在就接受这项强制性的 AI 职场培训——不然的话。

我想做一个大家共同使用的应用，汇总并分享我们在处理繁重行政事务、官僚体系的烂摊子、卡夫卡式的退订迷宫、错综复杂的保险门户、错误收费、被拒理赔、看起来让人困惑的会员方案等等事上花费了多少时间和精力。

我尽可能清楚、尽可能详细地描述了一个仪表盘，记录我们集体负担的这种“垃圾”到底有多大、有多广。用户会把生活中的烦人事件记下来，填写自己花了多少时间、烦得多严重，以及本来更想做些什么。每次提交都能得到一份“多巴胺式”奖励：一句励志又不失反讽的反抗格言，以及一张小猫、小狗，或者小黑猩猩宝宝的照片。然后我会训练 Claude 生成一段“更广泛语境”——解释这个烦人事件如何嵌入了更大的系统性“烂摊子”模式——并自动写一封给相关监管机构的投诉信。

Claude 开始犹豫。并非第一次，我担心我的“vibe”会直接变成一张报错页。我模糊地想起在 Reddit 论坛上看到的一些建议：\“先把电脑和代码的原理学清楚。\”“考虑去上 Harvard 的 CS50。\”“不要先去学 AWS 或服务器，先用像 Kuberns 这样的工具。\”我开始担心，vibe coding 可能只是“炫技鸡汤”：当然任何人都能做，但你先得先掌握哈佛级别、涉及几十种编程语言和云平台的知识。

这种担心只持续了大约三个“Kuberns”这么长的瞬间。Claude 停止了思考，随后开始展开分析，并说到本质上它必须承认这是个了不起的想法：\“这是个非常棒的点子——真正实用，有清晰的目标，而且对一个真实问题有很好的幽默感。我们先在深入之前，先把全盘情况讲实在。\”

又问了几个澄清问题后，我面前已经是一个真正的界面了。“Log Incident”和“Dashboard”两个标签页还没法用，我们还没把条目保存到任何地方，而且我还得继续给Claude讲解更完整的上下文。不过，一个在线应用的雏形已经出现了。

接下来的一个小时，我一直在把问题细节都理顺。Claude可以处理一些修正，也有一些是它让我来做的。我什么也不太懂，只是在照着它的指令操作（尽管我也是下达指令的人）。但我们确实在稳步推进，帮助是——自信、让人安心、说得清楚——总是唾手可得：

我：我已经完成了上面第三步，但到第四步就开始困惑了。下面是我点击“设置”后看到的界面截图。

CLAUDE：好消息，你的位置是对的，而且已经很接近了。不过我可以看到，Supabase自从我写那份说明后已经更新了他们的界面。你现在看到的是新的API密钥页面，和我描述的那版略有不同……

这种体验很像搭建一个复杂的乐高作品：你并不知道每一根带肋护套或每一片外鼓式挡泥板具体有什么作用，但只要你完全按说明一步一步来，作品就会被搭出来。

Linus Torvalds、Steve Wozniak、Bill Gates：我猜这些人也只能借了妈妈的电脑一段时间后就不行了。几个小时后，我告诉 Claude，我们很快再继续。

我开开心心地开车回家——这种兴奋让我想起我二十多岁时短暂的电弧焊接时期。我简直不敢相信自己，一个普通人，居然能做出这样的事！对于我每天能快速完成的那些网站和应用，它们一直对我来说都像谜一般，像是由一群神秘修士建起的金字塔。突然间，我变成了一个建金字塔的人。

我并不孤单。佛罗里达有人最近做了一个叫 Stratus 的东西——一种吉他效果器，玩家可以用英语直接描述音效，比如“给我一个带温暖 Mellotron 感的抖动颤音（wobbly tremolo）”，它就能生成声音。别处有个叫 Justin 的人做了一个 Plywood Cutting Visualizer，输入板材尺寸就能直接得到切割方案。还有人做了 MIXCARD，把你的 Spotify 歌单变成了实物明信片。对某些人来说，在某个下午，想法与创造之间的壁垒实际上已经消失了。

但这也是个问题所在：当任何有点心血来潮的人都能自己做应用时，会发生什么？那些环境、政治和经济方面的担忧又再次卷土重来——而这次，伴随着一个新的担忧。

在疫情前，我就开始请朋友来家里参加我称作“Admin Night”的仪式。这个仪式的想法是，与他人同行，狠下心清理你个人堆积的“垃圾”，把烦人的杂务一次性处理掉。我发现，如今的许多行政性任务其实是昨天地解决方案的残留——那些曾经承诺能提升效率或整理记忆的系统，后来要么坏了、要么过期了、要么开始按月收取 14.99 美元。我就在想，AI 会不会不同，还是我只是在帮它帮着制造更多“垃圾”？

几天后我又去看妈妈。我的应用已经有了进展，但最后这段路似乎花了和前面 90% 的工作同样长的时间。我需要在 GitHub（存放代码）、Supabase（存储用户的“垃圾记录”）和 Netlify（部署应用）上创建账号——每一项都很直观，却也都是出错点。比如我把 API Key 暴露在公开的 GitHub 仓库里。Claude 发现了这个问题，并把密钥移到了安全位置。接着又要考虑安全性。由于 Instagram 算法已经“懂”了我这个新爱好，我看到一个给应用开发者的提示词，教你如何让 Claude 进行安全审计。果不其然，我们发现用户提交的文本会未经清理直接插入页面 HTML，这意味着任何人都能把恶意代码伪装成公司名称提交，并在所有访问者的浏览器里执行。很容易修复。

这些任务都很琐碎，却都能完成：在各个服务之间来回传递凭据，点“Deploy”，看着某个环节失败，把报错贴回 Claude，再重复一遍。更像是组装，而不是工程开发——有点像拼装宜家床垫。之后我又去妈妈家三次，在她的胫骨恢复到足够稳固、可以试着用拐杖之前，我的应用已经可以试运行了。

我把爸爸叫到电脑前，让他讲了讲这个电话菜单系统，并把他的回答打进了应用程序——我发现，这一过程对他来说本身就有一种发泄的感觉。

每次我打电话预约医生，都得听一大堆选项提示，而这些选项几乎都不是来预约的病人真正会选的，比如最先出现的就是“如何发送传真”的信息。再加上界面上那句菜单最近改动了的提示也不太可信。整个系统看起来远没有发挥出应有的效率，而一遍又一遍地听着这段录音，时间久了会让人越听越发狂。

他在这上面花了多少时间？三小时。烦躁程度有多高？十分制的三分。那他更愿意做什么？做园艺。

我爸爸的提交最后得到了乌苏拉·勒古恩的一句引言作为奖励——“我们生活在资本主义中，它的力量似乎势不可挡；而国王的君权神授也曾是如此。”——配图是一只小腊肠犬在小溪边打盹。

于是我的自动上下文生成器就开始运转起来：

自动电话系统通常会根据通话量数据或行政管理上的便利性来优先排列选项，而不是按用户意图，所以才会出现传真发送——这个大概只是少数来电者会用到的遗留选项——排在了很可能带来最大来电量的预约功能之前。

而且，它还补充说，公司完全可以通过一些简单的调整来修复那个可疑的“最近菜单选项已更改”提示，这些调整“除了要花点人力时间去落实外几乎不费钱，但前提是把用户操作摩擦当作一个真正值得解决的问题来对待”。

我父亲虽然对我会操作电脑这件事本身就感到眼前一亮，但他对我这番“换位思考式”的说法尤其印象深刻。然而，Claude 和我之间仍然还有一项自动化内容审核流程要一起制定。现在，在用户提交内容出现在公开仪表板之前，需要经过一轮相当严格的筛选。（想顺利通过可不容易，CrotchGoblin69 这个账号最清楚。）两天后我把链接发到了我的 Admin Night 社群；我们存在的原因在于垃圾问题具有群体性，任何真正的解决办法也必须是群体性的。很快大家开始提交关于 Audible 信用政策、Hulu 重复计费、以及为购买女儿舞会门票所需的登录“体操”的投诉。一位成员丹妮尔把这整个过程比作“申诉式拦截网”。另一位艾米告诉我，这就像“一个朋友来你身边听你在诉说最近的分手，然后趁你洗澡时，强烈建议你创建一个新的约会资料页，并刻意避开你刚刚抱怨过的所有内容。”看到应用里的其他条目后，她又补充道：“这提醒人们，问题不在你本人，而在于系统。”

互联网或许能以某种有意义的方式重分配权力这一想法，从一开始就一直在给科技巨头及其帮凶提供掩护，而我并没有幻想，只要一股业余编程热就能从这群“浊流”手里夺回我们的时间和主导权。尽管如此，我那次业余代码突发的尝试已经在 Netlify 上线了。据我所知，它可能明天就会崩溃。但它确实存在——在曾经只有困惑和挫败的地方，出现了一个共享的公民账本。浊流靠在黑暗中榨干我们，并假定每个人浪费的时间加起来并不算什么来生存。其实是算得上的。我有一个粗糙的数据库来证明这一点。

也许我已经内化了 Claude 那些空洞的“你可以做到”的断言，但我被少量充满希望的 vibe 所激发的可能性深深打动。在更值得期待的手中，这些工具现在能做什么、正在做什么，根本无法预料。或许这一次，菜单选项真的可能会改变。
