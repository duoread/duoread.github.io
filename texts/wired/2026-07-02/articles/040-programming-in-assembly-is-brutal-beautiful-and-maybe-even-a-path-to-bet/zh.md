# 汇编语言编程既残酷、又优雅，甚至可能是一条通往更强大 AI 的路

Section: Gregory Barber
Date: 2025-10-13T07:00:00-04:00
Rubric: 无论你的芯片是在运行一款复古电脑游戏，还是最新的 DeepSeek 模型，它都会因为你使用它的母语而给予回报。

《过山车大亨》并不是 1999 年最时髦的电脑游戏。但是如果你看穿像素表层——摇摇欲坠的设施、饥渴交加又到处呕吐的人群（以及清洁工在他们身后一路拖着拖把）——深入代码层面，你会看到一种几乎接近疯狂的近乎偏执的匠心。该游戏的唯一开发者克里斯·索耶（Chris Sawyer）用汇编语言写完了全部内容。

像 Python、Go 或 C++ 这样的某些编程语言被称为“高级语言”，因为它们的写法有点像人类语言，由指令和表达方式组成，甚至可以写得像一段押韵朗诵。一般来说，编译器会把这类软件转换为机器真正读取的内容：由 1 和 0 组成的代码块（或许是十六进制），来告诉真实晶体管该如何工作。汇编语言是“低级语言”中最低的一层，与机器的本地方言几乎一一对应。它是直接对硬件编程。从汇编语言构建一款复杂的电脑游戏，像是在用剪下来的猫毛编织挂毯。

为什么会有人这么做？我最近在索耶的故乡——苏格兰——问过他。他告诉我，效率是其中一个原因。在 1990 年代，高级语言的工具链还不够完备。编译器慢得令人抓狂，调试器也很糟糕。索耶可以通过使用 x86 汇编来避开这些问题——这也是英特尔芯片的通用语言。

我们都知道这并不是他真正的理由。真正的原因是热爱。在转向过山车题材前，索耶用汇编语言写过另一款游戏《运输大亨》。这款游戏让玩家掌控一个城市的道路、铁路车站、跑道和港口。我想象索耶像一位模型火车爱好者——铺设每一段轨道、手工缝接人工草坪，每个细节都是一次选择，也是一项苦差事。为了将这些精心打造的像素从位图送到屏幕上，索耶不得不榨取芯片的全部潜能。“《过山车大亨》之所以诞生，是因为我熟悉了可能实现的边界”，他告诉我。

在这个数字技术极其丰富的时代，仍然强调“在约束中工作”似乎有些异类——如今训练一次 AI 算法里的单个函数调用就可能调动上百万块 GPU。用汇编则不同，机器只会准确执行你要求它做的一件事，且只执行这一件事；而且，正如许多程序员吃过的苦头所证明的那样，即使你要求的是错误的，它也只会忠实地做对。汇编语言就是这样“残酷而优美”。它要求你把你的意思说得精确无误。

我对汇编的创作者是有亏欠的。他们本来希望把事情变得更简单，而不是更难。我想，他们大概厌倦了反复装上打孔卡、在那些蒸汽朋克般庞然大物上拨动开关。或许他们曾梦想有一个像我们现在这样的世界：只需极少提示，计算机就能做很多事。

第一门汇编语言由凯瑟琳·布斯（Kathleen Booth）在20世纪40年代创建，尽管她经常没有得到应有的认可，真是讽刺。但它几乎看不起来像“语言”。“代码”仍然替代“代码”：为了让机器执行一个操作，比如在机器码里写“0,0111”，你要改用一串字母和符号，再由一类新软件——汇编器——把它们翻译成二进制。没过多久，指令就逐渐变得更接近日常表达，比如“MOV”这种更易读的助记符。

要掌握汇编，就要了解 CPU 本身——它能够做什么，更重要的是，它不能做什么。芯片的物理结构、AND 与 XOR 等逻辑门之间电路的实际布线方式，决定了它的工作方式。它的功能其实非常基础，把指令拆解为几个基本步骤：从内存取数并放入一个临时的小格子——我们称之为寄存器；在那儿解码；再做一些操作，比如比较两个值或把它们相加；然后再把结果送回内存。

随着芯片的发展，汇编语言出现了新的方言。把第一个人送上月球的代码就是汇编——它是专为只有一款芯片开发的，即阿波罗11号导航计算机。如果你想阅读会发光猫（Furby）的泄露源代码，你需要精通 6502。要破解你的 Ti-83 计算器，你则需要 z80。先学会一种芯片的语言——比如说 Intel 的 x86——再转去用 Arm，就像在贝鲁特学习阿拉伯语后，想在突尼斯或喀土穆勉强交流。祝你好运。

我在大学时作为一个“逃离数学”的人学习了 x86 汇编。我的同学似乎很喜欢 Java 那些单调的咒语般的写法，但我却爱上了汇编这场逻辑游戏。失败很容易发生，但只要你去看电路和寄存器，就能解释这些失败。正是这种看似不成体系的简单指令，让我有一种了不起的掌控感；我也很清楚这种掌控感有多脆弱。你想把这些字节放那里——不，是那里，在那个寄存器里、那些电容里。记住这个。忘掉那个。反复推敲，把每个“零件”一件件地打磨出来。

确实，现在在日常编码工作中使用汇编已经没有太大意义了。高级语言的效率很高，它们的抽象几乎总是更合适。甚至连汇编的发明者也转向了其他方向；Booth 在 1990 年代的最后论文之一，利用神经网络来将海豹和它们的吠声匹配起来。Sawyer 也同样转了方向。他最近一直在玩家庭自动化——比如灯光、温度传感器、音响系统等——用树莓派和 Python 来写代码，起初他告诉我这让他“感觉很别扭”。但即便在那块小处理器上，它也能干好这项工作。

然后，像 DeepSeek 这样的技术出现了，提醒人们：人类仍然能与硬件沟通得更好。就在今年，做出这些极其高效 AI 模型的这家中国公司颠覆了“AI 进步只能靠更多芯片和更多能耗”这一叙事。汇编是一个意外的关键原因。DeepSeek 的工程师在 Nvidia 芯片的底层精确地控制每一颗机器，在恰当的时机把数据从 32 位压缩到 8 位——以牺牲精度换取效率。观察者都惊呆了。难道真的能做到吗？DeepSeek 的工程师重新拿起了许多人早已遗忘的一门艺术。

我也有过同样的感受：2023 年，DeepMind 的研究人员教会了一台机器 x86 汇编语言后，又让它改进 C 语言里长期存在的 `sort()` 函数。AI 做出了一些奇怪、反直觉的选择，在寄存器之间进行怪异跳转，最终精确地减少了一个步骤。也许只节省了几分之一毫秒，但如今这套新算法已经被正式采纳后，这样的优化每天会发生无数次。

对我来说，这提醒了我：我们人类创造了这些机器，即便它们看起来似乎正朝着超出我们理解范围的复杂性不断螺旋上升，它们仍然受我们掌控。我们总能让它们运行得更好。就像 Sawyer 在回忆他最近那次用 Raspberry Pi 进行的居家编程实验时所说的那样。这也许只是他的想象，但他觉得显示有些卡顿。如果可以，他会重写那段代码。他说。可惜的是，Sawyer 和那台机器并没有使用同一种汇编语言。
