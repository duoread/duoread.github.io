# 我只是个普通人。普通人真的能“凭感觉”写代码吗？

Section: Chris Colin
Date: May 18, 2026 6:00 AM

把我推向技术未来的那只狗“又矮又壮”。这就是我妈妈在今年早些时候它在城市公园把她给撞到之前先注意到的唯一特征：矮、胖、跑得很快，甚至快得足以把她的右胫骨撞断。不过先不说她了。先说说这件事在我生活里引发了什么：在成功地“学了二十多年都没学会”任何编程之后，我马上就要尝试我的第一个软件开发项目了。

如果你也被一只又矮又壮的狗撞断过母亲的小腿骨，你就知道接下来会接踵而至的是哪些小挫败。比如说，我父亲花了几个小时在电话自动语音树里反复转接，想把我妈妈的医疗护理处理好。你会觉得这些让人抓狂的电话在大局里有多重要吗？不算什么大事。但那只该死的狗偏偏在一个技术上特别有意思的时刻发威：这是人类历史上第一次，问题不必严峻到什么程度，也能把严肃的工具召唤出来。

只要硅谷开始兜售“零摩擦的未来”，我们这些普通人就一直是它的被动顾客——在应用商店里刷来刷去，希望总有人替我们把需要的东西做出来。AI 和它的民主化拍档“氛围编程（vibe coding）”就这样闯了进来。如果这个承诺成立，那么我们忽然就可以自己做应用，不管有多小众、多琐碎，都不需要编程功底。我们只要对着那些烦人的点做个指示，一整套大型语言模型、代码生成器和开发环境就会立刻排成一列回应。

小众又琐碎？那就是我！别人用 vibe coding 来做简历审核器、库存追踪器和自动助理，以提高工作效率，而我的目标完全不同。过去几年里，我在个人和职业层面都特别沉迷于政策研究里常说的“泥淤（sludge）”这个概念：一股日益高涨的小型行政负担浪潮，它似乎在越来越定义我们今天的生活方式——并且侵蚀我们把事情真正做完的能力。比如处理保险、把保险和你的医生挂钩、追踪航空里程、或是处理孩子学校的门户网站。比如要对一笔账单提出异议，或取消一个你差点都忘了还在续费的流媒体服务。每一件事都像是对我们时间的一次独立进攻。但它们并不独立。它们更像一张同一套共生菌丝网络上，迥然冒出的不同蘑菇。

从某种程度上说，这是一个“校准”问题。虽然更严重的问题——至少在理论上——会引起注意：立法、新闻报道、参议院听证会；但那些更小的麻烦，琐碎到不值得诉讼，就只好成为生活中的常态。历史的车轮或许会向公平方向弯曲，但在与一美元银行手续费作斗争时，它却更偏向等待音乐。

也正是在这种情况下，“氛围编程”的幻想吸引了我的注意。这些烦人环节并非复杂系统的偶然副产物；它们常常就是一种设计。一个让人困惑的入口页面、一通被悄悄中断的电话、一个故意模糊到让人不愿继续推进的流程。在大规模上，它们更像是政策，而不是程序漏洞。我设想的应用会把这一现象摆出来，让这些义务的累计压力更难被忽视。我希望你脑海里浮现的画面，是一片轻轻颤动的蘑菇地。

我妈妈虽然没有一双结实的双腿，却靠一份 Claude Pro 订阅来补齐短板。过去几年我反复提醒她关注人工智能的环境、政治和经济影响，但最近这个星期天，我把这些都放在一边，开车去看她。聊了会儿胫骨的事后，我打开了她的电脑，开始“释放氛围”。

现在就去完成这项必修的人工智能职场培训——否则就会有后果。

我想做一个大家共同使用的应用，把我们花在应对沉重行政事务、官僚化手续、Kafka 式退订迷宫、复杂到难解的保险门户、错误扣费、理赔被拒、令人困惑的会员方案等事情上所耗费的时间和精力都集中收集并共享出来。

我尽可能清楚、尽可能详细地写下了这个想法：做一个仪表盘，用来记录我们共同遭遇“烂泥潭”的规模和范围。用户可以把自己生活中的烦心事件记下来，填写花了多少时间、烦到什么程度、以及本来想做什么。每提交一条记录，系统会用一句鼓舞人心的抗争名言和一张小猫、小狗或小黑猩猩宝宝的照片来给用户一个“多巴胺式”鼓励。我会让 Claude 自动生成一段“更宽的背景”——讨论这次挫败如何嵌入更大规模的系统性烂摊子里——以及一封给相关监管机构的投诉信。

Claude 在思考。并不是第一次，我担心我的“vibe”最后只会变成一片报错页。我模模糊糊想起在 Reddit 上看到过的一些建议：先把电脑和编程的基本原理学会；先看看哈佛的 CS50；与其先去学 AWS 或服务器，不如用 Kuberns 这种东西。我开始担心，vibe coding 其实就是“石头汤”：看起来人人都能做，但先得先懂哈佛水准的、几十门编程语言和云平台。

这种担心只持续了大约三秒 Kuberns。Claude 不再犹豫，转而去展开它本质上不得不承认很棒的方案： “这是个很好的点子——真有用，而且有明确目标，也能用幽默来对准一个真实的问题。让我先给你一个诚实的现状说明，再开始推进。”

过了几次澄清问题之后，我终于盯着一个真正的界面看了。 “记录事件（Log Incident）”和“仪表盘（Dashboard）”两个标签还没起效，我们还没把条目保存到任何地方，而且我还需要再教 Claude 更广泛的上下文内容。但一个线上应用的雏形已经出现了。

接下来一个小时我一直在把问题一一抚平。有些修正是 Claude 能直接做的，有些则需要我来做。我其实一窍不通，只是在机械地照着指令执行（而且同时也是那个下达指令的人）。但我们还是稳步推进，帮助始终就在身边——自信、安抚、清晰——并且触手可及：

我：我完成了上面第 3 步，但第 4 步让我有点糊涂。下面是我点击“设置”后看到的截图。

Claude：好消息是，你来对地方了，而且已经很接近了。只是我看到 Supabase 自从我写这些说明后已经更新了界面。你现在看到的是他们新的 API 密钥界面，这个界面和我之前描述的有一点点不同……

这种体验很像在组装一个复杂的乐高作品：你不知道每一个“Ribbed Hose”或“Flared Mudguard”单独有什么作用，但只要你严格按说明书一步步来，最终这个东西就能被搭出来。

Linus Torvalds、Steve Wozniak、Bill Gates：我猜这些人也只不过把妈妈的电脑借来用了那么一阵子。两个多小时后，我告诉 Claude，我们很快会继续。

我开车回家的时候像在发疯一样兴奋——这种兴奋感我在二十多岁时短暂做过电弧焊那段时间里也有过。真不敢相信我，一个普通人，竟然能做到这个！对我来说，每天随手点开的那些网站和应用，一直都神秘得像由难以理解的祭司团建造的金字塔。突然之间，我变成了金字塔建造者。

我并不孤单。最近有位在佛罗里达的人做了一个叫 Stratus 的东西，一个吉他效果器，可以让玩家用纯英语描述音效——“给我一个发抖的颤音，带点温暖的 Mellotron 感”——然后它就能生成。别的地方，一个叫 Justin 的人做了 Plywood Cutting Visualizer，只要输入板材尺寸，就能返回切割方案。还有人做了 MIXCARD，把你的 Spotify 播放列表变成实体明信片。对某类人来说，在某个下午，想法与创作之间的鸿沟，实际上已经消失了。

但这也是个隐患：当任何有点冲动想法的人都能自己做应用时会怎样？那些环境、政治和经济层面的担忧又重新猛烈涌现，还带来了一种新的担忧。

在疫情前，我开始邀请朋友来参加我称作“行政之夜”的一种惯例。这个想法是和他人一起集中处理自己的各种杂务。 我注意到，现在很多行政工作其实是昨日技术方案的残留——那些号称能提高生产力或整理记忆的系统，后来要么坏掉了，要么过期了，要么开始每月收费14.99美元。人工智能会不一样吗？还是我只是在帮着制造更多“数字垃圾”？

几天后我又去看了妈妈。我的应用已经在推进，但最后这段路程似乎和前90%花了差不多的时间。我需要在GitHub（存放代码）、Supabase（存放用户的“杂务记录”）、Netlify（部署应用）里各开一个账号，每一步都很直接，却也都可能出错。比如，我把API密钥暴露在公开的GitHub仓库里。Claude发现了它，并把密钥移到安全位置。接着还得考虑安全问题。我的Instagram推荐算法已经注意到我这个新爱好，给我推了一个面向应用开发者的提示词，解释如何“命令”Claude做一次安全审计。果然，我们发现用户提交的文本会未经清理直接插入页面HTML，这意味着任何人都可以把恶意代码当作公司名字提交，并在每个访问者的浏览器里执行。这个问题很容易修复。

这些任务很琐碎但都能完成：在各项服务之间传递凭证，点击“Deploy”，看着某个步骤失败，把报错贴给Claude，再重复一遍。是组装，不是工程——更像拼装宜家床。之后我又去了妈妈家三次，到她的胫骨恢复到可以试着用拐杖时，我的应用已经可以试运行了。

我把爸爸叫到电脑前，采访了他关于这个电话语音树的看法，并把他的回答打进了这个应用——我注意到，这对他来说本身就是一种宣泄。

每次我打电话预约看病时，都得听一长串选项，这些选项几乎不会是打电话来预约的病人会选的，比如第一个就是发送传真信息的说明。再加上最近菜单选项已经变动了的说法也不太可信。整个系统看起来远没达到该有的效率，而反复坐着听这段录音，时间一长就会让人觉得越发疯狂。

他花了多少时间来踩这些流程？三小时。你说有多烦？十分制里三分。那他更愿意做什么？种花种草、园艺。

我父亲的提交内容获得了乌苏拉·K·勒古因的一句名言奖励——“我们生活在资本主义中。它的力量似乎无可逃避。当年王权神授也如此。”——并配上了一只在溪边打盹的小腊肠狗。

从那里开始，我的自动上下文生成器就投入工作了：

自动电话系统通常会按通话量数据或行政上的便利性优先列出选项，而不是按用户真实意图排列，这也是为什么传真传输——通常只是少数来电者在用的遗留选项——会被放在比预约挂号功能更前面，尽管后者才可能产生大部分来电。

更重要的是，它继续说，公司其实可以用一些很简单的调整来修复“最近更改了菜单选项”这一可疑说法，“只需要别人花点时间来执行，花销不大，但前提是把用户摩擦感当作一个值得真正解决的问题来对待。”

我父亲在我会操作电脑这件事上本来就很佩服，但他尤其被这种视角转换能力惊到。然而，我们和 Claude 之间仍需要一套自动化内容审核流程。现在，用户投稿在显示在公开仪表盘之前，要先经过一套相当严谨的过滤机制。（想通过它，真不容易，CrotchGoblin69。）两天后我把链接发给了我的 Admin Night 社区；我们会聚在一起是因为这种“垃圾投诉”问题具有群体性，真正的解决方案也必须是群体性的。很快他们开始提交抱怨：Audible 的信用积分政策、Hulu 的重复扣费、给女儿买舞会门票时需要做的登录体操。有位成员丹妮尔把整件事比作“申诉式围捕网”。另一位成员艾米说，这就像“一个朋友来陪你在浴室里哭诉你最近的分手，她在外面认真听着，等你淋浴的时候，她却强烈帮你推荐一个全新交友账号，专门避开你刚才提到的所有问题。”她补充道，看着应用里的其他条目，“让我想起了一件事：这不是你个人的问题，而是系统问题。”

互联网或许能在实质上重新分配权力——这一想法长期以来都为大科技公司及其帮凶提供了掩护，我并不天真，认为一阵“业余编程”就能把时间与主动权从那些垃圾内容生产者手里夺回来。尽管如此，我这次“业余编程”的尝试已经上线了 Netlify。依我猜，它明天可能就会挂掉。但它的确存在了——这是一本共同的公民账本，之前那里只有挫败感。垃圾内容产业得以生存，正因为它在黑暗中耗尽我们，并假设我们每个人被浪费的时间加起来并不构成意义。事实并非如此。我有一份简陋的数据库来证明这一点。

也许我已经把 Claude 那种空洞的“励志鸡汤”内化了，但我被那一点点希望感真的能激起的效果深深触动。换成更有分量的掌舵者时，这些工具还可以做什么、目前正在做什么，都没法预估。也许这一次，真正能改变的也许真的是“菜单选项”本身。
